# ReactJS là gì?

![ReactJS](/static/media/reactjs.jpg)

ReactJS được thiết kế như mã nguồn mở và tập trung vào trải nghiệm của người dùng. Tuy nhiên ReactJS được tính là một library của JavaScript nhưng cách vận hành của nó không khác gì mấy so với framework. Khi sử dụng ReactJS, đa phần code của lập trình viên được gọi là "React". Cho nên, React được tính là một framework bởi vì nó điều khiển phần lớn các code của lập trình viên. Nói cách khác, nếu Angular là một framework trong ứng dụng, thì ReactJS có thể được mô tả như framework trong giao diện. Khác với một vài framework khác, thì ReactJS phù hợp với đa số công nghệ mà lập trình viên dùng. Tương tự như các đối tượng trong phần mềm lập trình hướng đối tượng (OPP) vậy, ReactJS cho phép lập trình viên Front End có thể tạo ra các module để tiện và có thể quản lý riêng biệt các module đó, tối ưu hoá cho việc tìm kiếm và hiển thị dữ liệu. Điều làm cho lập trình viên thích thú khi sử dụng React Framework là tính tương thích khi hiển thị ở cả máy chủ và máy tính của người ta.

Tuỳ theo độ bảo mật yêu cầu, lập trình viên có thể điều chỉnh cho phép một vài thành phần nhất định hiển thị trên máy tính của người dùng. Song song với việc sử dụng ReactJS dùng cho phát triển Web, lập trình viên Front End có thể sử dụng thêm một vài framework, mã nguồn mở khác rất phổ biến là React Native (có chung cơ sở React nhưng dùng cho việc phát triển Mobile Apps.)

Các dự án React (ReactJS) / React Native có thể được tái sử dụng nhiều module thay vì được phát triển lại từ đầu. Đây là một thuận lợi để các nhà phát triển phần mềm tập trung phát triển website với trải nghiệm người dùng và giao diện bắt mắt.
