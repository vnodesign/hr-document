Git là công cụ quản lý mã nguồn phổ biến nhất hiện nay. Cho dù bạn là một lập trình viên hay một người không có chuyên môn kỹ thuật, Git sẽ giúp bạn với các tính năng hợp tác như theo dõi lỗi, quản lý nhiệm vụ và wikis.

Git đóng vai trò quan trọng trong tổ chức để triển khai các phương pháp DevOps. Nhìn vào sự phổ biến của Git trong DevOps, rõ ràng là sẽ có nhiều cơ hội việc làm trong tương lai gần. Theo một báo cáo từ Grand View Research, thị trường DevOps toàn cầu dự kiến sẽ đạt 12,85 tỷ đô la vào năm 2025, tăng trưởng với tỷ suất CAGR là 18,60%. Dưới đây là một số câu hỏi phỏng vấn Git hàng đầu sẽ giúp bạn vượt qua cuộc phỏng vấn.

## Git là gì và tại sao nó được sử dụng?

Git là một hệ thống quản lý mã nguồn phân tán miễn phí và mã nguồn mở. Nó được phát triển bởi Linus Torvalds vào năm 2005 cho dự án Linux kernel, và sau đó trở thành một trong những công cụ phổ biến nhất cho việc quản lý mã nguồn trong các dự án phần mềm.

Git được sử dụng để theo dõi các thay đổi trong mã nguồn và phối hợp các phiên bản khác nhau của mã nguồn giữa nhiều nhà phát triển. Nó cho phép người dùng lưu trữ mã nguồn trên một máy chủ từ xa hoặc trên máy tính cá nhân của họ, tạo ra các nhánh và ghép nhánh (merge) các thay đổi vào nhánh chính một cách an toàn. Git cũng cung cấp các tính năng như theo dõi lỗi, quản lý nhiệm vụ, wiki và nhiều tính năng khác để hỗ trợ việc phát triển phần mềm và quản lý dự án.

Với Git, các nhà phát triển có thể làm việc độc lập trên các phiên bản mã nguồn khác nhau và dễ dàng hợp nhất các thay đổi để phát triển phần mềm một cách hiệu quả. Nó cũng giúp giảm thiểu các xung đột mã nguồn và hỗ trợ các kỹ thuật phát triển phần mềm như Agile và DevOps.

## Các câu hỏi phỏng vấn cơ bản về Git

### 1. Git là gì?

Git là một hệ thống quản lý phiên bản được sử dụng để theo dõi các thay đổi trong các tập tin máy tính và giúp phối hợp công việc giữa nhiều người trong dự án và theo dõi tiến độ trong thời gian. Nó là một công cụ giúp quản lý mã nguồn trong phát triển phần mềm.

Git hỗ trợ cả các lập trình viên và người dùng không có chuyên môn kỹ thuật bằng cách giữ bản ghi của các tập tin dự án của họ. Nó cho phép nhiều người dùng làm việc cùng nhau và xử lý các dự án lớn một cách hiệu quả.

![Git là gì](/static/interview/git.png)

### 2. Bạn hiểu thế nào về thuật ngữ "Hệ thống quản lý phiên bản" (Version Control System)?

Hệ thống quản lý phiên bản (VCS) ghi lại tất cả các thay đổi được thực hiện trên một tập tin hoặc một tập dữ liệu, vì vậy một phiên bản cụ thể có thể được gọi lại sau này nếu cần.

Điều này giúp đảm bảo rằng tất cả các thành viên trong nhóm đang làm việc trên phiên bản mới nhất của tập tin.

![Hệ thống quản lý phiên bản](/static/interview/server.jpg)

### 3. GitHub là gì?

GitHub là một nơi lưu trữ trên Internet cho việc quản lý phiên bản và phát triển phần mềm, sử dụng công cụ Git.

### 4. Hãy liệt kê một số dịch vụ lưu trữ Git phổ biến.

Các dịch vụ lưu trữ Git phổ biến bao gồm:

1. GitHub: Là nơi lưu trữ mã nguồn phổ biến nhất và cũng được sử dụng rộng rãi để chia sẻ mã nguồn và hợp tác trên các dự án phần mềm.
2. GitLab: Cung cấp các tính năng quản lý dự án và Git repository, bao gồm kiểm soát quyền truy cập và tích hợp liên kết với các công cụ phát triển phần mềm khác.
3. Bitbucket: Cung cấp các tính năng quản lý dự án, kiểm soát phiên bản và tích hợp liên kết với các công cụ phát triển phần mềm khác.

### 5. Các loại hệ thống quản lý phiên bản khác nhau

Các loại hệ thống quản lý phiên bản khác nhau bao gồm:

1. Local Version Control System: Lưu trữ phiên bản trực tiếp trên máy tính cá nhân của người dùng.
2. Centralized Version Control System: Sử dụng một máy chủ trung tâm để lưu trữ tất cả các phiên bản và cho phép nhiều người dùng truy cập cùng một lúc.
3. Distributed Version Control System: Được sử dụng bởi các nhóm phát triển phần mềm để quản lý các phiên bản phân tán, mỗi người dùng có thể lưu trữ phiên bản trên máy tính cá nhân của mình và chia sẻ chúng với nhau.

Một số ví dụ về hệ thống quản lý phiên bản là: CVS, SVN (Subversion), Git, Mercurial.

### 6. Sử dụng GIT đem lại những lợi ích gì?

Sử dụng GIT mang lại những lợi ích sau đây:

1. Quản lý phiên bản dễ dàng: GIT cho phép bạn quản lý các phiên bản của mã nguồn của dự án một cách dễ dàng và tiện lợi. Bạn có thể xem lại lịch sử thay đổi, so sánh các phiên bản và quay lại phiên bản trước đó nếu cần thiết.
2. Cộng tác: GIT cho phép nhiều người cùng làm việc trên cùng một dự án, cho phép các thành viên cập nhật và đồng bộ hóa code với nhau.
3. Khôi phục dữ liệu dễ dàng: Nếu mã nguồn bị mất hoặc bị hỏng, GIT cho phép bạn khôi phục lại các phiên bản trước đó của dự án.
4. Tiết kiệm thời gian và công sức: Với GIT, bạn có thể lưu trữ và quản lý code trên một nền tảng đáng tin cậy, giúp tiết kiệm thời gian và công sức cho các nhà phát triển.
5. Đảm bảo tính toàn vẹn của dữ liệu: GIT cho phép bạn kiểm tra tính toàn vẹn của dữ liệu và đảm bảo rằng các thay đổi được áp dụng đúng cách.
6. Dễ dàng tích hợp: GIT có thể tích hợp với các công cụ phát triển phần mềm khác, giúp tối ưu hóa quy trình phát triển và cải thiện hiệu suất của dự án.

### 7. Sự khác biệt giữa Git và GitHub là gì?

| Git                                                                | GitHub                                                       |
| ------------------------------------------------------------------ | ------------------------------------------------------------ |
| Git là một phần mềm                                                | GitHub là một dịch vụ                                        |
| Git có thể được cài đặt trực tiếp trên hệ thống local của máy tính | GitHub được lưu trữ trên web                                 |
| Git cung cấp một giao diện desktop được gọi là git GUI             | GitHub cung cấp giao diện desktop được gọi là GitHub Desktop |
| Nó không hỗ trợ tính năng quản lý người dùng                       | Cung cấp quản lý người dùng tích hợp sẵn                     |

### 8. Kho lưu trữ Git là gì?

Một kho lưu trữ Git là một kho chứa tất cả các tệp và thư mục cần thiết cho dự án của bạn, cùng với lịch sử các sửa đổi của chúng. Kho lưu trữ này được quản lý bởi Git và được lưu trữ trên máy tính cá nhân hoặc trên các dịch vụ đám mây như GitHub, GitLab hoặc Bitbucket. Nó cho phép các nhà phát triển làm việc trên các phiên bản khác nhau của cùng một dự án cùng một lúc và quản lý các thay đổi của mình một cách hiệu quả.
